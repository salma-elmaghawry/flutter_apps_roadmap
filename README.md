Salma is building **5 Flutter apps to publish on Google Play** to strengthen her portfolio and land
her first Flutter developer job. Full roadmap: `~/.claude/plans/ok-now-i-see-witty-sketch.md`.

Direction she chose (as of 2026-07-08): **UI/UX-craft-forward** (every app has a signature
animation), **Firebase/Supabase** backends, **MENA / Arabic-first** (RTL + Arabic, Paymob/Fawry
gateways), **5 focused MVPs**. Each app must close a distinct CV skill gap.

The 5 apps (build order): 1) **Sawtak** voice notes (audio, speech-to-text, AI summaries),
2) **Rasseed** expense tracker (ML Kit receipt OCR, charts, home widget), 3) **Fahim** AI study
companion (flashcards from PDFs, spaced repetition, Rive) — prioritized because her YouTube/Cairo
Coding School audience gives real users, 4) **Fasila** real-time study/mentor chat (realtime, FCM,
deep links), 5) **Souq** e-commerce + Flutter Web admin (payments, multi-platform, charts).

**Why:** Her portfolio Projects section is thin (only Shtiwy + Chainly) with nothing live on a store;
the bar she's aiming for is the reference portfolio (`reference .pdf` = Khaled ElMetwaly, 15+ shipped
apps) framed Problem→Approach→Impact.
**How to apply:** As each app ships, add it to the portfolio Projects section in `index.html` using
the existing `.project-card` pattern with a Play Store link (add a Google Play icon to the
`data-brand` set in [[]]`script.js`). Skills currently missing from her CV: real-time+FCM, audio/STT,
ML Kit OCR, payments/IAP, Flutter Web+admin, localization+RTL, biometrics, home widgets, Rive/Lottie,
CI/CD→Play. See also portfolio work in this repo.
